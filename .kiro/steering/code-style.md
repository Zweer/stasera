# Code Style

## TypeScript

- Strict mode, no `any` — use `unknown` and narrow
- Explicit return types on exported functions
- ES modules only (`"type": "module"`)
- `async/await` over `.then()/.catch()`
- Prefer `const` over `let`, never `var`
- Use early returns to reduce nesting
- Prefer named exports over default exports (except page/layout)
- No unused code, no commented-out code

### Naming
- **camelCase** for variables, functions, methods
- **PascalCase** for components, types, interfaces
- **UPPER_SNAKE_CASE** only for env vars
- **kebab-case** for file names

### Import Order
```typescript
// 1. External packages
// 2. Internal aliases (@/*)
// 3. Relative imports
// 4. Types (with `type` keyword)
```

## Dependencies

- Pin exact versions (no `^` or `~`)
- Keep dependencies up to date
- Prefer native Web/Node APIs when possible

## Components

- One component per file, named export matching filename
- Props interface defined and exported above the component
- Use `cn()` for conditional Tailwind classes
- `components/ui/` is auto-generated by shadcn — never edit manually
